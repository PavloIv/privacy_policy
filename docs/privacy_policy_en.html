Privacy Policy for “Komunalka”

Last updated: March 17, 2026

This Privacy Policy explains what data is processed by the “Komunalka” mobile application (the “App”), for what purposes, how long data is stored, and how users can manage their data.

1. Who we are

“Komunalka” is an app for tracking utility tariffs, meter readings, and payments, with local storage and Firebase synchronization.

Privacy contact: grafmk1523@gmail.com

2. What data we process

2.1 Account data (Google Sign-In / Firebase Authentication)

  • Firebase user identifier (uid)
  • Google account email
  • Google profile name (if available)

2.2 Data you enter in the App

  • Nickname
  • Address
  • Selected app language
  • Tariff names and tariff values
  • Readings, amount/difference, total sum, date, paid status

2.3 Technical data

  • FCM token for push notifications (Firebase Cloud Messaging)
  • Minimum technical data required for Firebase services

3. Where and how data is stored

  • On device: local database (SQLite/Drift) and app preferences.
  • In cloud (optional via sync): Firebase Cloud Firestore under your account.

Synchronization is used for:

  • backup;
  • restoring data on a new device;
  • using the same account data on multiple devices.

4. Why we use data

Data is used only for:

  • user authentication;
  • tracking and displaying utility data;
  • cross-device synchronization;
  • local/push notifications;
  • proper app functionality.

We do not sell personal data to third parties.

5. Data sharing with third parties

Data is shared only with infrastructure providers needed for app features:

  • Google Firebase Authentication
  • Google Cloud Firestore
  • Firebase Cloud Messaging

These services process data under their own policies:

  • Firebase Data Processing and Privacy
  • Google Privacy Policy

6. User control

You can manage data in the App via:

  • profile editing;
  • adding/editing/deleting tariffs and readings;
  • manual synchronization;
  • logout.

Deleted records are marked as deleted and synchronized so they are not restored on other devices.

7. Data retention and deletion

  • Local data remains on your device until removed by you or cleared on logout/app reset.
  • Cloud data in Firebase remains as long as needed for your account unless deletion is requested.

To request full deletion of cloud-related data, contact: grafmk1523@gmail.com.

8. Security

We rely on standard Firebase security mechanisms, including authentication and Firestore access rules, to restrict data access to the account owner.

No transmission or storage method is 100% secure.

9. Children

The App is not intended for children under 13 (or other age required by local law). We do not knowingly collect data from children.

10. Policy updates

We may update this Policy from time to time. The latest version always includes the “Last updated” date.
